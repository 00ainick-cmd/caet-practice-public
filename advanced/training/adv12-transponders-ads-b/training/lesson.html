Section 8: Transponder & ADS-B Systems

⏱ ~45 minutes 📖 Engage → Explore → Explain → Elaborate → Evaluate 🛠 Hands-on + drill
Engage

Why should you care about this?

~4 min

The transponder is the single point of failure that stands between an aircraft and its legal right to operate in controlled airspace. Every radar target on a controller's scope at an ARTCC or TRACON is painted by an interrogation-reply pair between ground radar and a transponder bolted to the belly of an airplane. When the transponder is wrong - wrong altitude, wrong squawk, wrong ICAO 24-bit address, stale NACp - the controller's separation data is wrong. Separation is the load-bearing function of the National Airspace System. A technician who signs off a bad 91.413 test has not just committed a paperwork error; they have injected a bad data source into the separation system that keeps airplanes from hitting each other. This is why 14 CFR 91.413 is biennial and uncompromising, why the test procedures in Part 43 Appendix F are prescriptive to the kilohertz, and why transponder work is one of the most regulated tasks in the entire Part 145 repair station.

On January 1, 2020, the FAA's ADS-B Out mandate under 14 CFR 91.225 went into effect for Class A, B, and C airspace (and above the ceiling of Class B/C up to 10,000 ft MSL within the lateral boundaries of the Mode C veil). Overnight, every transponder installation in US-registered aircraft operating in controlled airspace became a compliance artifact - not just an avionics box, but a federally-mandated surveillance source with performance requirements written into 14 CFR 91.227. An installation that meets 91.225 spatially but fails 91.227 on NACp, NIC, SIL, or SDA is a violation regardless of whether anyone gets caught. The FAA's Compliance Monitor tool (public web portal) analyzes every ADS-B broadcast in the US and flags non-performing installations. Your shop's reputation rides on those reports, because they are emailed to the aircraft owner with the tail number in the subject line.

Discover the terminology

~8 min

Avionics work has its own vocabulary. Master these terms before procedures, because the oral-board examiner expects precision, not approximations.

Mode A

Transponder mode that replies to ground interrogation with a 4-digit octal squawk code (0000-7777, 4096 possible codes). Identifies the aircraft to ATC but carries no altitude. Governed by TSO-C74d.

Mode C

Transponder mode that replies with the squawk code PLUS the pressure altitude encoded via Gillham (Gray) code from the altitude encoder. Always reports altitude referenced to 29.92 in Hg, regardless of Kollsman setting. Required in Class A/B/C airspace and within the Mode C veil.

Mode S

Selective interrogation mode that addresses individual aircraft using a unique 24-bit ICAO address. Carries altitude, squawk, flight ID, and extended data fields. Governed by TSO-C112d and RTCA DO-181E. Required for TCAS II to function and mandatory for ADS-B Out via 1090ES.

1090ES (Extended Squitter)

ADS-B Out data link operating on the Mode S 1090 MHz frequency. Uses the Extended Squitter message format to broadcast position, velocity, and ID every 0.5-1 second. Governed by RTCA DO-260B and TSO-C166b. Mandatory for aircraft operating above 18,000 ft MSL (Class A).

UAT (Universal Access Transceiver)

ADS-B Out data link operating on 978 MHz, permitted only below 18,000 ft MSL. Lower-cost alternative to 1090ES, popular for GA. Governed by RTCA DO-282B and TSO-C154c. Also carries FIS-B weather and TIS-B traffic as ADS-B In services.

ADS-B Out

Automatic Dependent Surveillance-Broadcast transmit function. The aircraft broadcasts position (from GPS), altitude, velocity, heading, and ID automatically and continuously - no interrogation required. Performance rules in 14 CFR 91.227.

The theory you need

~12 min

Memorizing steps isn't enough. The evaluator asks why. Here is what you need to understand.

8.2.1 — Transponder Modes

A transponder is a secondary surveillance radar (SSR) responder. It listens on 1030 MHz for interrogations from ground radar or TCAS, and replies on 1090 MHz with coded data. The mode determines what data is in the reply. Modern installations are universally Mode S, which supersedes Mode A and Mode C while remaining backward-compatible with them. Understanding how each mode is structured is required to interpret test results and troubleshoot ATC squawks.

  • Mode A replies carry only the 4-digit octal squawk code set by the pilot - 12 data bits total. No altitude, no identity beyond the code. Governed by TSO-C74d and ICAO Annex 10 Volume IV.
  • Mode C replies add the pressure altitude in Gillham (Gray) code, 11 bits encoded in 100-foot increments, range -1000 ft to +126,700 ft. The reply is a Mode A frame with altitude bits replacing the zero slots.
  • Mode S replies use a completely different waveform: Pulse Position Modulation at 1 Mbps, 56 or 112 bits per reply, addressed to a specific 24-bit ICAO address. Supports selective interrogation, preventing the 'garbling' that occurs when multiple aircraft reply to the same Mode A/C interrogation.
  • Mode S Extended Squitter (ES) is what makes 1090ES ADS-B Out possible. The transponder transmits ES messages unsolicited every 0.5-1 second carrying position, velocity, identity, and status. This is 'squitter' because no interrogation triggers it.

8.2.2 — ADS-B Out

ADS-B Out is the aircraft's self-reported surveillance broadcast. Unlike the transponder, which is interrogated and responds, ADS-B Out broadcasts unsolicited. The aircraft computes its position from an onboard GPS source, packages it with altitude, velocity, identity, and integrity metrics, and transmits it on either 1090 MHz (1090ES) or 978 MHz (UAT). 14 CFR 91.225 mandates ADS-B Out in defined airspace; 14 CFR 91.227 defines the performance the installation must deliver.

  • 91.225 applies to: Class A, B, and C airspace; Class E airspace above 10,000 ft MSL (excluding airspace at/below 2,500 ft AGL); above the ceiling and within the lateral boundaries of Class B/C airspace up to 10,000 ft MSL; and the ADIZ. Effective date January 1, 2020.
  • 91.227 performance: NACp >= 8, NIC >= 7, SIL >= 3, SDA >= 2, NACv >= 1. Latency from measurement to broadcast <= 2.0 s (uncompensated) or 2.0 s (compensated if designed to reduce latency). Broadcast rate 1 Hz for position, velocity, and heading.
  • 1090ES is required above 18,000 ft MSL (Class A). UAT (978 MHz) is permitted below 18,000 MSL only. An aircraft operating in both regimes needs 1090ES or dual-link; many GA aircraft with UAT avoid Class A entirely and only carry 1090 on the transponder as backup Mode S.
  • GPS/WAAS position source must be TSO-C145c/C146c (SBAS-capable). Bare TSO-C129 GPS is NOT compliant for ADS-B Out after Jan 2020 because it cannot deliver required NIC/NACp. Garmin GNS 430W/530W and GTN 650/750 are compliant; GNS 430/530 (non-W) are NOT.

8.2.3 — Transponder Test Requirements

14 CFR 91.413 requires transponder testing every 24 calendar months. The test is not optional and not negotiable. Part 43 Appendix F prescribes the exact tests, tolerances, and pass/fail criteria that a certified shop must meet. The test must use calibrated test equipment and be recorded in the aircraft maintenance records. Separate from 91.413, ADS-B Out installations added by modification require an installation check per AC 20-165B, and any transponder replacement triggers a re-check of the Mode S address and ADS-B broadcast.

  • 91.413 (transponder test) and 91.411 (altimeter and altitude reporting system test) are cross-referenced. An encoder correlation check appears in both - in practice, shops usually combine the two tests when the aircraft is on the pitot-static jig.
  • Who can perform: certified repair station appropriately rated, properly rated manufacturer, or holder of Continuous Airworthiness Maintenance Program (CAMP). Not a standard IA or A&P. This is a 'return to service' item requiring a repair station certificate.
  • Part 43 Appendix F tests: (a) radio reply frequency 1090 +/- 3 MHz; (b) suppression (no reply to interrogations at the SLS level); (c) receiver sensitivity (-73 dBm typical); (d) reply delay 3.0 us +/- 0.5 us; (e) reply jitter < 0.1 us; (f) reply rate (up to 500 Mode A/C replies per second); (g) Mode A code correctness; (h) Mode C altitude encoding accuracy (+/- 125 ft of the altimeter at 29.92); (i) Mode S address verification; (j) power output (typically > 125 W for Class 1).
  • Test equipment: Aeroflex/VIAVI IFR-6000, Tel-Instrument/Intermarine T-49/T-50, or VIAVI AVX-10K. All are calibrated annually and maintain 37 CFR / NIST traceable standards. Due-date sticker on the unit is mandatory in your test documentation.

8.2.4 — Altitude Encoder

The altitude encoder is a purpose-built pressure transducer that converts barometric pressure altitude into Gillham (Gray) code - an 11-bit parallel output that the transponder reads and encodes into its Mode C reply. The encoder always reports altitude referenced to 29.92 in Hg. This is NOT a design flaw; it is a deliberate standard so that ATC computers can apply the local altimeter setting uniformly across all aircraft. An encoder that reports Kollsman-corrected altitude would break the entire ATC altitude picture.

  • Gillham code is a reflected binary (Gray) code in 100-ft increments from -1000 ft to +126,700 ft. Each encoder output line corresponds to a specific bit: D2, D4, A1, A2, A4, B1, B2, B4, C1, C2, C4 on 11 open-collector pins.
  • The encoder must correlate with the primary altimeter within +/- 125 ft per Part 43 Appendix E (altimeter test) and Appendix F (transponder test). Measured with altimeter Kollsman window set to 29.92 in Hg.
  • Common encoders: ACK A-30, Ameri-King AK-350, Trans-Cal SSD120, Sandia Aerospace SAE5-35, Trig TA50. Integrated air data computers (Aspen Avionics EA100, Garmin GDC 74A, Sandia SAC7-35) replace discrete encoders and feed the transponder with RS-232 or ARINC 429 altitude serial data.
  • Encoder power: typically 14 V or 28 V DC. Warm-up time 3-5 minutes before altitude output is stable. An encoder that has been unpowered and is tested too quickly will read wrong - always power the aircraft for the stabilization interval before pitot-static testing.

Task focus: Task 8.4.1 — Transponder Modes and Emergency Codes

Read the procedure once all the way through before you touch a tool. Note how each step has a verification built in — the procedure is self-checking.

  1. Identify the transponder panel and make/model (e.g., Garmin GTX 345, L3 Lynx NGT-9000, Trig TT31). Note the mode-selector layout: a rotary knob or softkey labeled OFF / STBY / ON (or ALT) / ALT (or Mode C) / TEST.
  2. Power up aircraft avionics bus and wait for the transponder self-test to complete (3-5 seconds on solid-state units, up to 60 seconds on older cavity units).
  3. Select STBY and explain: 'In standby, the transponder has power and has performed its internal BIT but will not reply to interrogations. Used on the ground before taxi to avoid cluttering ground surveillance radar.'
  4. Select ON (some units label this Mode A). Explain: 'In ON, the unit replies to Mode A interrogations with the squawk code but does not reply to Mode C altitude interrogations. Legacy mode - rarely used today because modern Mode S units in ALT serve both purposes.'
  5. Select ALT (Mode C / Mode S operational). Explain: 'In ALT, the transponder replies to all interrogations - Mode A for squawk, Mode C for altitude, and Mode S for selective interrogation. This is the normal in-flight setting and is required in controlled airspace.'
